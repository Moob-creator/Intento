# Intento - 智能 Todo 桌面应用 PRD

**Version:** V3.1
**Date:** 2026-02-09
**Status:** Draft

---

## 一、产品定位

Intento 是一款**面向个人用户的智能任务管理桌面应用**，通过简单的文字或图片输入，快速创建和管理任务，让用户专注工作而不遗漏关键事项，并自动生成不同时间维度的工作总结档案。

**核心价值：**
- 零配置、零学习成本，输入即任务
- 自动理解上下文，无需手动分类
- 自动生成日/月/季/年总结，回顾产出

---

## 二、目标用户

**主要用户画像：**
- 工作节奏快、信息碎片化的知识工作者（产品、研发、设计等）
- 使用桌面办公，需要快速记录待办事项
- 希望减少任务管理的心智负担
- 想要定期回顾自己的工作成果

**不适用人群：**
- 需要团队协作、项目管理的场景
- 需要复杂看板、甘特图的项目经理

---

## 三、核心功能

### 3.1 多模态输入解析

**输入方式：**
- 纯文本："下午记得发测试包"
- 图片/截图：转发聊天截图，自动提取任务信息

**自动识别：**
- 用户意图（创建新任务 vs 更新已有任务）
- 任务标题（动作+对象）
- 时间信息（"明天"、"下周一"、"下午三点前"）
- 关键信息（责任人、优先级等）

**实现方式：**
- 通过云端 AI API（OpenAI/Claude）进行自然语言理解和 OCR 识别

---

### 3.2 智能上下文理解

系统自动从最近的交互中提取语境，增强识别准确性：
- "刚才那个 bug" → 关联最近讨论的任务
- "文案改一下" → 可能属于同一主题

**用户无需：**
- 创建项目
- 管理分类标签
- 上传背景文档

上下文理解完全隐式，在后台自动完成。

---

### 3.3 任务确认与管理

**确认界面：**
系统解析后展示简洁的确认卡片：
- 任务标题
- 描述（可选）
- 截止时间
- 是否关联已有任务

**用户操作：**
- 直接添加
- 简单修改后添加
- 忽略/取消

**任务管理：**
- 列表视图（To Do / Doing / Done）
- 简单搜索
- 任务编辑（标题、描述、截止时间）

**不支持：**
- 项目管理
- 看板视图
- 复杂依赖关系
- 多维度筛选

---

### 3.4 智能提醒

**A. 到期提醒**
- 任务接近截止时间时，桌面通知提醒

**B. 每日回顾提醒（可选）**
- 默认每天 18:00 推送：
  - 今日未完成的任务
  - 即将到期任务
- 用户可关闭或调整时间
- 若应用未运行，下次启动时补充提示

---

### 3.5 自动总结与档案

**总结维度：**

| 维度 | 内容 | 生成时机 |
|------|------|----------|
| **每日总结** | 完成/进行中/新增/风险任务 + 自然语言总结 | 每天固定时间，离线则下次启动补生成 |
| **月度总结** | 按主题聚合完成任务 + 关键事件 + 跨月未完成 | 每月初自动生成 |
| **季度总结** | 主要成果 + 工作方向聚合 + 节奏变化 | 每季度初生成，可用于 OKR Review |
| **年度总结** | 全年任务量 + 关键成就 + 类别占比 + 高峰/低谷 + 年度文案 | 每年初生成，帮助复盘成长轨迹 |

**档案结构：**
```
总结中心（Timeline）
├─ 2026 年度总结
├─ 2025 Q4 总结
│   ├─ 12 月总结
│   ├─ 11 月总结
│   └─ 10 月总结
├─ 每日总结（最近两周）
...
```

用户可自由浏览历史档案。

## 四、典型使用场景

### 场景 1：截图转任务
1. 用户收到微信截图："这个 Bug 明天修一下"
2. 复制图片到 Intento
3. 系统自动识别：标题"修复 Bug"，截止"明天"
4. 用户确认后添加

### 场景 2：快速语音转文字输入
1. 用户语音输入："下午记得发测试包"
2. 系统识别：标题"发测试包"，截止"今天下午"
3. 用户确认添加

### 场景 3：更新任务状态
1. 用户输入："登录崩溃的问题解决了"
2. 系统识别是已有任务的更新
3. 提示用户标记为完成

### 场景 4：回顾产出
1. 用户想看"最近做了什么"
2. 打开总结中心
3. 查看每日/月度总结

## 五、技术架构

### 5.1 技术选型建议

| 模块 | 技术方案 |
|------|----------|
| **桌面框架** | Electron / Tauri（建议 Tauri，体积小）|
| **前端框架** | React / Vue 3 |
| **本地数据库** | SQLite（存储任务、总结档案）|
| **AI 能力** | OpenAI API / Claude API |
| **图片识别** | GPT-4 Vision / Claude 3 Vision |
| **定时任务** | Node-cron / 系统级定时任务 |

### 5.2 数据存储

**本地存储结构：**
- `tasks.db`：任务表（id, title, description, status, deadline, created_at, updated_at）
- `summaries.db`：总结表（id, type, date, content, generated_at）
- `context.db`：上下文缓存（最近 N 条交互记录）

**数据安全：**
- 数据完全存储在用户本地
- 仅在调用 AI API 时传输必要信息
- 敏感信息（如截图中的个人隐私）需脱敏处理后传输

## 六、非功能性需求

| 指标 | 要求 |
|------|------|
| **响应速度** | 输入到展示确认界面 ≤ 3 秒 |
| **学习成本** | 首次使用即可完成任务创建 |
| **稳定性** | 每日/月度总结最终必须生成（允许延后补生成）|
| **数据安全** | 本地存储，用户数据不外泄 |
| **界面简洁性** | 避免复杂结构，保持轻量 |

## 七、MVP 范围

### ✅ MVP 包含
- 文本 & 图片输入解析
- 自动任务识别 + 二次确认
- 基础任务管理（列表视图 + 状态管理）
- 到期提醒
- 每日总结
- 月度总结（简化版）

### ❌ MVP 不包含
- 项目/迭代功能
- 多人协作
- 看板视图
- 深度数据可视化
- 自定义总结模板

## 十、风险与限制

| 风险 | 应对措施 |
|------|----------|
| **AI API 成本** | 控制单次调用 token 数，缓存常见识别结果 |
| **离线场景** | 提示用户网络异常，暂存任务手动录入 |
| **误识别** | 强制二次确认流程，降低误操作 |
| **总结生成失败** | 补生成机制，保证档案完整 |
