智能 Todo 工具（轻量版 + 自动总结）

版本	V3
状态	草稿
最后更新	2025-12-01


⸻

1. 产品定位

一款通过简单输入（图片 / 文字 ）即可快速创建并管理任务的智能 Todo 工具。
用户无需配置、无需建立复杂结构，所有智能行为均在后台自动完成。

目标是让用户在工作繁忙时，避免遗漏，并能在不同时间点轻松回顾自己的产出。

⸻

2. 产品目标
	1.	减少任务遗漏：用户把碎片信息直接发送进来即可生成任务。
	2.	不打断工作流：尽量减少用户需要手动整理任务的工作量。
	3.	帮助用户回顾：提供每日、每月、每季度、每年的自动总结档案。
	4.	保持工具轻量：不增加用户负担，不引入复杂结构（如项目管理、知识库等）。

3. 用户场景

3.1 场景：通过截图创建任务

用户收到聊天截图：“这个 Bug 明天修一下”。
→ 转发截图到 Todo
→ 系统自动提取可执行内容
→ 用户确认后落为任务。

3.2 场景：通过自然语言快速添加

“下午记得发测试包。”
→ 自动识别截止时间 & 动作
→ 用户确认
→ 任务添加。

3.3 场景：更新任务

“登录崩溃的问题解决了。”
→ 系统识别是某个已存在任务的完成状态
→ 提醒用户是否标记为完成。

3.4 场景：查看近期成果

用户希望看“最近一段时间我做了什么”。
→ 打开总结中心
→ 查看每日、月度或年度总结。

⸻

4. 核心功能需求（Feature Requirements）

⸻

4.1 多模态输入解析

用户通过以下方式发送输入：
	•	图片 / 截图
	•	文本

系统自动识别：
	•	用户意图（是否要创建或更新任务）
	•	任务标题（动作 + 对象）
	•	关键时间（例如“明天”、“下周一”、“下午三点前”）
	•	关键信息（例如“让小李处理”）

目的：让用户无需手动填写大量表单。

⸻

4.2 自动上下文理解（无配置）

系统自动从最近的交互中提取必要语境，用于增强识别效果，例如：
	•	“刚才那个 bug” → 关联用户最近讨论的任务
	•	“文案改一下” → 可能属于同一个主题

用户无须：
	•	创建项目
	•	管理分类
	•	上传文档
	•	做任何配置

上下文理解是完全隐式的。

⸻

4.3 任务生成与二次确认

系统生成任务后，展示一个简洁的确认界面：
	•	任务标题
	•	描述（可选）
	•	截止时间
	•	是否关联已有任务（如是更新）

用户可：
	•	直接添加
	•	简单修改后添加
	•	忽略任务

此确认步骤用于避免误识别，是唯一的“可见智能流程”。

⸻

4.4 任务管理

提供基础、轻量的任务管理能力：
	•	列表视图
	•	任务状态（To Do / Doing / Done）
	•	简单搜索
	•	任务编辑（标题、描述、截止时间）

不提供：
	•	项目管理
	•	看板、泳道
	•	复杂依赖关系
	•	多维度筛选

工具保持轻量、不打扰。

⸻

4.5 智能提醒

类型：

A. 到期提醒

任务快到截止时间时，推送提醒。

B. 每日回顾提醒（可选）

每天固定时间（默认 18:00），推送：
	•	今日未完成的任务
	•	即将到期任务

若提醒时间点应用未运行，则在下次启动时集中提示当天未完成和即将到期任务，不保证严格准点推送。

用户可关闭或调整时间。

⸻

4.6 自动总结与档案（Daily / Monthly / Quarterly / Yearly）

系统自动生成并存档总结，不需要用户手动维护。

4.6.1 每日总结

生成内容包含：
	•	今日完成的任务
	•	今日处理中的任务
	•	今日新增
	•	今日风险（逾期或即将到期）
	•	一段自然语言总结

每天固定时间尝试自动生成并存档；若该时间段应用未运行，则在下次启动或后台检查时根据历史任务补生成缺失总结，确保时间轴完整。

⸻

4.6.2 月度总结

每月生成如下内容：
	•	本月完成任务（按主题或类别自动聚合）
	•	本月关键事件
	•	跨月未完成任务
	•	月度自然语言总结

用于回顾整月产出。

⸻

4.6.3 季度总结

每季度自动生成：
	•	本季度主要成果
	•	大方向工作聚合（基于任务主题自动识别）
	•	工作节奏与产出变化
	•	可用于 OKR Review 的总结内容

⸻

4.6.4 年度总结

包含：
	•	全年任务完成总量
	•	关键成就
	•	不同类别任务占比
	•	全年高峰/低谷
	•	一段年度自评文案（AI 生成）

帮助用户查看全年成长轨迹。

⸻

4.6.5 总结档案

系统会以时间轴（Timeline）形式保存所有总结：

总结中心
 ├─ 年度总结
 ├─ Q4 总结
 │   ├─ 12 月总结
 │   ├─ 11 月总结
 │   └─ 10 月总结
 ├─ 每日总结（最近两周）
 ...

用户可自由浏览。

5. 非功能性需求（NFR）
	•	响应速度：从输入到展示确认界面不超过 3 秒
	•	简洁性：界面与功能保持轻量，避免复杂结构
	•	低学习成本：用户第一次使用即可完成任务创建
	•	数据安全：确保用户内容不会被错误展示给其他用户
	•	稳定性：每日/每月总结需确保最终生成（允许因离线延后，通过补生成机制保证档案完整）

⸻

6. 范围（Scope）

6.1 MVP 包含
	•	输入解析（文本 & 图片）
	•	自动任务识别 + 二次确认
	•	基础任务管理
	•	到期提醒
	•	每日总结
	•	月度总结（简化版）

6.2 MVP 不包含
	•	项目/迭代功能
	•	复杂协作能力
	•	多人共享任务
	•	看板视图
	•	深度数据可视化
	•	自定义总结模板

⸻

7. 示例交互

示例：用户转发截图

内容：“请在周三前修下这个 crash。”
→ 系统识别
→ 标题：修复 crash
