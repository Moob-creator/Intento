# macOS 通知问题修复指南

## 问题描述

通知只出现在通知中心，没有桌面横幅弹出。

## 原因分析

macOS 有两种通知样式：
1. **横幅** (Banner) - 短暂显示后自动消失
2. **提醒** (Alert) - 持续显示直到用户关闭

默认情况下，应用的通知样式可能设置为"横幅"，且如果没有声音，可能会被静默处理。

---

## ✅ 解决方案 1: 修改系统通知设置（推荐）

### 步骤 1: 打开系统设置

1. 点击屏幕左上角  菜单
2. 选择"系统设置" (System Settings)
3. 点击"通知" (Notifications)

### 步骤 2: 找到 Intento

1. 在左侧应用列表中找到"Intento"
2. 如果找不到，先运行一次 Intento 并发送测试通知

### 步骤 3: 调整通知样式

**推荐设置:**
- ✅ **允许通知** (Allow Notifications) - 开启
- ✅ **通知样式** → 选择 **"提醒"** (Alerts)
  - 或选择"横幅" (Banners) 也可以，但提醒更显眼
- ✅ **通知横幅显示时间** → "持久" (Persistent) 或 "临时" (Temporary)
- ✅ **在锁定屏幕上显示** (Show on Lock Screen) - 开启
- ✅ **在通知中心显示** (Show in Notification Center) - 开启
- ✅ **标记图标** (Badge app icon) - 开启
- ✅ **播放提示音** (Play sound for notifications) - 开启

## 🔍 故障排除

### 问题 1: 找不到 Intento 通知设置

**原因:** 应用首次运行时未请求通知权限

**解决:**
1. 运行 Intento
2. 发送一次测试通知
3. 系统会弹出权限请求 → 点击"允许"
4. 然后就能在系统设置中找到 Intento

### 问题 2: 通知仍然没有横幅

**检查清单:**
- [ ] 系统设置 → 通知 → Intento → 允许通知 (已开启)
- [ ] 通知样式设置为"提醒"或"横幅"
- [ ] 播放提示音已开启
- [ ] 勿扰模式 (Do Not Disturb) 已关闭
- [ ] 屏幕不在全屏应用中（全屏时横幅可能不显示）

### 问题 3: 开发模式下通知正常，打包后不正常

**原因:** Bundle Identifier 问题

**解决:**
修改 `src-tauri/tauri.conf.json`:
```json
{
  "identifier": "com.intento.todo"  // 改掉 .app 后缀
}
```

## 📱 通知样式对比

| 样式 | 显示方式 | 持续时间 | 适用场景 |
|------|---------|---------|----------|
| **提醒** | 屏幕右上角固定显示 | 直到用户关闭 | ⭐ 重要提醒（推荐） |
| **横幅** | 屏幕右上角短暂显示 | 5-10 秒后消失 | 一般通知 |
| **无** | 仅通知中心 | - | 不推荐 |

---

## 💡 最佳实践建议

### 针对任务提醒

1. **使用"提醒"样式** - 确保不会错过重要任务
2. **开启声音** - 听觉提醒更有效
3. **开启锁屏显示** - 即使屏幕锁定也能看到

### 针对每日总结

1. 可以使用"横幅"样式 - 不那么打扰
2. 可以关闭声音 - 更安静
3. 保留通知中心 - 方便稍后查看
