# 无忧记

## 动机
前段时间工作太忙，在开发的过程中有很多小的问题需要处理，脑子记不住全部的并且带来负担很大，就想着将需要处理的事情记录下来，做一个todo-list， 但是很多问题都是在聊天过程中给你口头说的，如果要记录下来需要稍微转化以下。
虽然花不了很长时间，但是很容打断手头上正在做的事情。还有一个就是每天忙来忙去，做了不少的事情，但是都没有记录下来，最后回顾的时候就要从记忆中疯狂找，所以我想可以从每日的todo-list中生成日报、周报之类的东西。

基于前面提到的两个需求，我想利用大模型来识别我的意图，快速的记录的当前问题和需要做的事情以及总结最近我在做的事情。同时作为一名客户端开发，正好可以开发一款桌面端的软件来练练手。

## 技术栈
这里准备选取rust+tauri+react 进行开发，利用框架可以很容易实现跨win和mac

## 大致的功能

- todo-list 这个这个软件基本的功能
- 对内容进行总结
- 需要有一个project（分组）的概念，在这个分组可以自定义上下文和相关的信息，比如：工作做的某个项目整体安排，ai会基于这个上下文更容易识别你的意图
